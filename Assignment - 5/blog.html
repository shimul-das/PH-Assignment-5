3. What is event bubble?

Event Bubbling is one characteristic these events share. I'll clarify what this action entails. The DOM has a concept known as event bubbling (Document Object Model). The event bubbles up (or you could say is transmitted or propagated) to the element's parent and grandparent elements in the DOM tree until it reaches the root element once an element receives an event. You can handle an event in a parent element rather than the actual element that received it thanks to the "Event Bubbling" behavior.

4. What is a callback function and why will you use it?

A callback function is a function that receives an argument from another function and is subsequently called from inside the outer function to finish a task or activity. The basic goal of a callback function is to free up the calling function from having to poll or check for a condition all the time by allowing the calling function to call back or tell the caller when a job has been finished or a condition has been satisfied. In event-driven programming, callback functions are frequently used. They are supplied to event handlers so that they can be invoked when a specific event takes place. They can also be supplied to asynchronous functions that will be invoked once the asynchronous process has finished in asynchronous programming.
